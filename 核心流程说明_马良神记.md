# 马良笔记（Maliang Notes）核心流程说明文档

## 产品概述

马良笔记是一款以 AI 驱动的碎片信息管理工具。超级大脑，智慧记忆。

当前默认支持五大记忆类型：
- 🍜 取餐码 — 外卖/咖啡取餐号
- 📦 取件码 — 快递取件信息
- 💰 账单 — 消费/收入记录
- 👕 服饰 — 衣橱管理
- 📝 随手记 — 临时备忘信息

更多类型可手动定义。

---

## 核心流程一：信息捕获与 AI 识别

### 流程描述

用户通过拍照或截图将碎片信息输入 App，AI 自动识别图片内容、提取结构化数据、归类为对应记忆类型。

### 三种入口

```
入口 A：系统分享
任意 App 截图 → 系统分享菜单 → 选择"马良笔记"
→ App 后台启动 AiProcessingService（前台服务）
→ 通知栏显示"AI 识别中…"进度条
→ AI Vision API 分析图片
→ 生成 1~N 条记忆记录（一图多记录）
→ 通知栏更新为识别结果

入口 B：快捷磁贴
下拉通知栏 → 点击马良笔记磁贴（QuickTileService）
→ 弹出 TilePickerActivity（相机/相册选择）
→ 选择图片后启动 AiProcessingService
→ 后续流程同入口 A

入口 C：App 内拍照
打开 App → 点击底部浮动按钮
→ ImagePicker 选择图片
→ 首页显示骨架屏加载动画（SkeletonListItem）
→ AI 识别完成后，新记忆以发光边框动画（AIGlowBorder）出现
```

### AI 识别流程（AiService.analyzeImage）

```
1. 图片压缩（Isolate 后台线程，避免 UI 卡顿）
   - 最大尺寸 1024px，JPEG 质量 85%
2. 调用 Vision API（支持自定义 API 地址/密钥/模型）
3. AI 多类型检测 + 二次验证防误分类
4. 一张图可同时识别多种信息类型
   例：外卖截图 → 取餐码 + 账单（两条独立记录）
5. 自动提取结构化字段：
   - 取餐码/取件码：码值、店名、快递公司、地址
   - 账单：金额、收支类型、分类（22种支出+8种收入）、商家、支付方式、时间
   - 服饰：名称、类型、颜色（hex值）、季节、品牌、价格、尺码
   - 随手记：标题、摘要、日程时间
6. 失败重试：3 次指数退避
```

### 通知上岛常驻（NotificationService）

```
使用 Google Live Update 接口，未完成的取餐码/取件码 → 通知栏大字显示码值
→ 支持"已取"按钮一键标记完成
→ 完成后自动从通知栏消失
```

## 核心流程二：首页信息流浏览

### 交互细节

- 列表项支持左滑删除，带触觉反馈（HapticFeedback）
- 新识别的记忆以 AI 发光边框动画渐入
- 滚动到边缘时触发弹性触觉反馈（ScrollEdgeHaptic）
- 分类 Tab 吸顶，滚动时固定在顶部
- 合集卡片按压缩放动画（scale 0.95）
- 支持平板横屏 Master/Detail 双栏布局（ResponsiveLayout）

## 核心流程四：账单汇总与 AI 财务分析

### 流程描述

首页点击"总账单"合集卡片 → 进入 BillSummaryPage


### AI 财务分析流程

```
1. 进入页面 → 自动触发 AI 分析
2. 构建账单上下文（支出/收入汇总、分类TOP5、商家频次、每日明细）
3. 调用 AiService.chatStream → 流式输出分析文字
4. AI 以朋友聊天口吻给出消费洞察
   例："这个月餐饮花了 ¥487，日均 ¥35，最常去肯德基和瑞幸"
```

### AI 对话交互（AiChatSheet）

```
点击"问 AI"按钮 → 弹出底部对话面板（渐变光效背景）

## 核心流程五：衣橱管理与 AI 穿搭推荐

### 流程描述

首页点击"我的衣橱"合集卡片 → 进入 WardrobePage

### AI 穿搭推荐流程

```
1. 进入衣橱页 → 自动触发
2. WeatherService 获取当前天气（温度、湿度、风速、城市）
   - 使用 Geolocator 获取位置
   - 调用 Open-Meteo API + Geocoding 反查城市
3. 构建衣橱摘要（名称、类型、颜色、季节，最多20件）
4. AiService.chatStream 流式生成穿搭建议（60-100字）
5. 解析 AI 回复中的【推荐：衣服名称】标记
6. 展示推荐衣物缩略图（可点击进入详情）
7. 支持"问 AI"进一步对话
```

## 核心流程六：合集管理

### 流程描述

用户可创建自定义合集，将相关记忆归组管理。

```
首页合集区域 → 点击"+"按钮
→ 弹出 CreateCollectionSheet
→ 输入合集名称 + 描述
→ 选择要加入的记忆
→ 保存

合集卡片 → 点击进入 CollectionDetailPage
→ 查看合集内记忆列表
→ 编辑名称/描述
→ 添加/移除记忆
```

### 内置合集

- 总账单：自动聚合所有账单类型记忆，进入 BillSummaryPage
- 我的衣橱：自动聚合所有服饰类型记忆，进入 WardrobePage

## 核心流程七：设置与数据管理

### 数据备份流程

```
导出：
所有记忆 + 图片 → 序列化为 JSON → 打包为 .maliang 文件 → 分享/保存

导入：
选择 .maliang 文件 → 解析 JSON → 恢复记忆数据 + 图片 → 刷新首页

## 关键交互设计特征

| 特征 | 实现方式 |
|------|---------|
| 一图多记录 | AI 单次识别可生成多条不同类型的记忆 |
| 零输入记账 | 拍照→AI自动提取金额、分类、商家、时间 |
| 通知栏常驻码值 | 未完成的取餐码/取件码大字显示，一键标记完成 |
| 流式 AI 回复 | chatStream 逐 token 输出，像朋友打字聊天 |
| 语音自然交互 | 2秒静默自动发送，无需手动点击 |
| 天气感知穿搭 | 实时天气 + 衣橱数据 → 个性化推荐 |
| 玻璃拟态 UI | BackdropFilter 毛玻璃效果 + 渐变光效 |
| 触觉反馈 | 滑动删除、按压缩放、边缘弹性均有震动反馈 |
| 响应式布局 | 手机单栏 / 平板双栏自适应 |
| Figma Squircle | 使用 figma_squircle 实现 iOS 风格平滑圆角 |
