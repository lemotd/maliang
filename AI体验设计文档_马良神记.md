# 马良笔记（Maliang Notes）AI 体验设计文档

## 一、场景选择：Android 手机碎片信息管理体验

现代人每天被大量碎片信息包围——外卖取餐码、快递取件码、支付账单、新买的衣服、临时备忘……这些信息散落在微信聊天、短信、支付宝、相册等十几个 App 中，需要时却找不到。

马良笔记选择了一个独特的切入点：**以"拍照/截图"为唯一入口，用 AI 自动理解图片内容，将碎片信息归类为五大记忆类型**——取餐码、取件码、账单、服饰、随手记，并在此基础上提供智能分析、穿搭推荐、财务洞察等增值服务。

本文档以 Android 系统为体验平台，分析用户在"碎片信息捕获与管理"这一场景下的痛点，以及 AI 如何从根本上重塑这一体验。

---

## 二、痛点分析

### 2.1 现状描述：用户当前管理碎片信息的操作路径

以几个典型场景为例：

**场景 A：外卖取餐**
```
1. 在外卖 App 下单，看到取餐码"A032"
2. 切到其他 App 做别的事
3. 到店后想不起取餐码，重新打开外卖 App
4. 在订单列表中翻找当前订单
5. 找到取餐码，告诉店员
6. 取餐后，这个信息再也用不到了，但订单页面依然堆积
```

**场景 B：快递取件**
```
1. 收到短信/App 推送："您的快递已到菜鸟驿站，取件码 12-3-4567"
2. 到驿站时翻找短信，在几十条垃圾短信中定位取件码
3. 如果有多个快递，需要逐条翻找不同的取件码
4. 取完件后，短信依然留在收件箱中，无法自动清理
```

**场景 C：日常记账**
```
1. 支付完成后，打开记账 App
2. 点击"记一笔"
3. 选择支出/收入类型
4. 在 20+ 个分类中找到对应分类
5. 手动输入金额
6. （可选）填写商家、备注、时间
7. 保存
→ 每笔消费重复以上步骤，一天 3-5 笔 = 15-35 步操作
```

**场景 D：衣橱管理**
```
1. 买了新衣服，拍照发朋友圈
2. 换季时想不起自己有哪些衣服
3. 出门前纠结穿什么，翻衣柜 10 分钟
4. 不知道已有衣服怎么搭配
5. 重复购买类似款式的衣服
```

**场景 E：临时备忘**
```
1. 看到一条有用的信息（会议时间、地址、WiFi 密码等）
2. 截图保存到相册
3. 需要时在几百张照片中翻找
4. 找不到，或者找到了但忘记截图的上下文
```

### 2.2 痛点定位

| 痛点类型 | 具体表现 | 影响的场景 | 严重程度 |
|----------|----------|-----------|----------|
| **信息分散** | 取餐码在外卖 App、取件码在短信、账单在支付 App、备忘在相册，没有统一管理入口 | 全部 | ⭐⭐⭐⭐⭐ |
| **手动录入繁琐** | 记账需要手动输入金额、选分类；衣橱需要手动标注颜色、尺码、季节 | 账单、服饰 | ⭐⭐⭐⭐⭐ |
| **时效性信息难找** | 取餐码、取件码是临时性信息，用完即弃，但在需要的那几分钟内却很难快速找到 | 取餐码、取件码 | ⭐⭐⭐⭐ |
| **缺乏智能关联** | 一张外卖截图同时包含取餐码和支付金额，但传统工具只能处理单一维度 | 取餐码+账单 | ⭐⭐⭐⭐ |
| **无主动提醒** | 快递到了不提醒取件，取餐码过号不提醒，账单超支不预警 | 取件码、取餐码、账单 | ⭐⭐⭐⭐ |
| **衣橱无感知** | 不知道自己有什么衣服、不知道怎么搭配、不知道今天天气适合穿什么 | 服饰 | ⭐⭐⭐ |
| **截图堆积无序** | 相册中大量截图混杂，无法按内容类型检索 | 随手记 | ⭐⭐⭐ |

**不必要的步骤**：
- 手动分类 → AI 应该自动识别图片内容并归类
- 手动输入结构化数据（金额、码值、尺码等）→ AI 应该从图片中直接提取
- 在多个 App 间切换查找信息 → 应该有一个统一的碎片信息中心
- 手动标记"已完成" → 取餐码用完、快递取完后应自动或一键归档

**让用户烦躁的环节**：
- 到店后翻找取餐码/取件码的焦虑感
- 每天重复性的手动记账操作导致放弃记账习惯
- 换季时面对满柜子衣服不知道穿什么的选择困难
- 截图保存了但需要时找不到的挫败感

### 2.3 量化感知

| 指标 | 数据估算 |
|------|----------|
| 目标用户群体 | 18-35 岁城市年轻人（外卖/快递/网购高频用户） |
| 日均外卖/咖啡订单 | 1-2 次（需要取餐码） |
| 周均快递量 | 2-5 个（需要取件码） |
| 日均消费笔数 | 3-5 笔（需要记账） |
| 月均购衣频次 | 2-4 次（需要衣橱管理） |
| 日均截图/拍照量 | 5-15 张（含各类碎片信息） |
| 传统记账 App 30 天留存率 | 仅 20-30%（因操作疲劳放弃） |
| 取餐码查找平均耗时 | 15-30 秒（在外卖 App 中翻找） |
| 取件码查找平均耗时 | 20-60 秒（在短信中翻找） |
| 痛点覆盖人群 | 中国城市 18-35 岁人群约 3 亿，其中 80%+ 有上述至少 3 个痛点 |

## 三、AI 改造机会

### 3.1 AI 能力映射

| AI 能力 | 应用场景 | 在马良笔记中的具体实现 |
|---------|----------|----------------------|
| **多模态感知（视觉识别）** | 一张图片自动识别多种信息类型 | Vision API 分析截图，同时提取取餐码"A032"和支付金额"¥35.00"，生成两条独立记录。支持识别取餐码、取件码、账单、服饰实物照、随手记等五大类型 |
| **自然语言理解（NLU）** | 语音记账、语音查询 | 用户说"刚才打车花了 35"→ 自动生成交通类账单；说"我这个月外卖花了多少"→ AI 实时分析并回答 |
| **上下文记忆** | 消费习惯学习、衣橱偏好记忆 | 记住用户常去的商家和消费区间，提高分类准确率；记住衣橱中所有衣物的颜色、季节、风格，用于穿搭推荐 |
| **意图预测** | 主动提醒、智能推荐 | 检测到未完成的取餐码/取件码 → 通知栏常驻提醒；检测到用户截图支付页面 → 主动询问是否记录 |
| **环境感知（天气+位置）** | 穿搭推荐、消费分析 | 获取当前天气（温度、湿度、风速）和城市信息，结合衣橱数据推荐今日穿搭；结合位置分析消费地点分布 |
| **数据分析与推理** | 财务洞察、消费预警 | 按月/按分类汇总支出，AI 生成个性化消费分析报告，发现异常消费模式并主动提醒 |
| **流式生成** | 实时对话体验 | AI 回复以流式 token 逐字输出，像朋友打字聊天一样自然，而非等待整段文字一次性返回 |

### 3.2 自然交互设计

**拍照/截图分享（核心入口）**：
- 从任意 App 分享截图到马良笔记 → AI 自动识别内容类型并提取结构化信息
- 通过 Android 系统下拉快捷磁贴（`QuickTileService`）→ 一键调起相机/相册，拍照后 AI 后台处理
- 在 App 内点击底部按钮 → 选择图片 → AI 识别

**语音交互**：
- 在账单汇总页或衣橱页点击"问 AI"按钮 → 自动进入语音监听模式
- 用户直接说话，AI 实时转文字（`speech_to_text`），2 秒静默后自动发送
- 支持切换到键盘输入模式，适应不同场景

**AI 主动预判**：
- 未完成的取餐码/取件码 → 通知栏常驻显示码值，支持一键"已取"标记完成
- 每日消费小结 → 晚间推送当日支出汇总
- 换季提醒 → 根据天气变化提醒用户查看衣橱中的应季衣物

### 3.3 个性化方案

**同样是"分享一张外卖截图"，AI 对不同用户的差异化处理**：

| 用户画像 | AI 行为差异 |
|----------|------------|
| **大学生小明**（高频外卖用户） | AI 识别出取餐码"A032"和金额"¥18.00"，自动生成两条记录。因为小明历史数据显示 90% 的外卖都是餐饮类，AI 直接归类为"餐饮"，不弹确认。通知栏大字显示取餐码方便到店取餐 |
| **白领小红**（注重消费管理） | 同样的截图，AI 除了生成取餐码和账单记录外，还主动提示："本周餐饮支出已达 ¥380，超出你设定的周预算 ¥350"。因为小红开启了预算提醒功能 |
| **宝妈李姐**（家庭采购为主） | AI 识别到商家是"盒马鲜生"，根据历史数据判断这可能是家庭采购而非个人餐饮，自动归类为"日用"而非"餐饮" |

**同样是"打开衣橱页面"，AI 对不同用户的差异化穿搭建议**：

| 用户画像 | AI 穿搭建议 |
|----------|------------|
| **北京用户，冬季 -5°C** | "今天降温明显，建议穿你那件黑色羽绒服，搭配深灰色休闲裤。围巾也别忘了带 🧣" |
| **广州用户，春季 22°C** | "今天温度刚好，那件白色衬衫配卡其色裤子就很合适，轻便又好看" |
| **同一用户，衣橱中运动装多** | AI 偏向推荐运动风搭配；如果衣橱中正装多，则偏向推荐商务休闲风 |

**个性化的技术实现路径**：
1. 基于用户历史记忆数据构建画像（高频商家、消费区间、衣橱风格偏好、取件频率等）
2. 在 AI Prompt 的 system message 中注入用户上下文（衣橱清单、天气信息、消费历史）
3. 分类置信度阈值根据用户纠错频率动态调整——纠错少的用户自动确认，纠错多的用户弹出确认
4. 衣橱推荐结合实时天气 API（Open-Meteo）和用户地理位置（Geolocator），给出因地制宜的建议

## 四、新体验完整描述

### 用户故事：小林的周末一天

> **角色**：小林，25 岁，互联网公司产品经理，使用马良笔记已两个月。他的马良笔记里已经积累了 87 条记忆——32 条账单、15 条取餐码、8 条取件码、12 件衣服、20 条随手记。

---

**上午 9:00 — 起床，纠结穿什么**

小林拉开窗帘，外面阴沉沉的。他打开马良笔记，滑到首页顶部的"我的衣橱"合集卡片，点进去。

衣橱页面顶部，AI 穿搭助手已经根据当前天气（上海，8°C，多云，风速 4m/s）和他衣橱里的 12 件衣服，给出了建议：

> "今天有点冷还有风，建议穿你那件藏青色卫衣，外面套黑色冲锋衣，下面配深色牛仔裤就行。别穿太薄，下午可能有雨 🌧️"

建议下方还展示了推荐衣物的缩略图——正是他衣橱里那件藏青色卫衣和黑色冲锋衣。小林点了点头，照着穿了。

---

**上午 10:30 — 取快递**

小林收到菜鸟驿站的短信，但他懒得翻短信。昨天他已经把快递通知截图分享到了马良笔记，AI 自动识别出了两个取件码：

- � 取件码 **12-3-4567**（顺丰，数码配件）
- 📦 取件码 **8-2-9901**（中通，书籍）

这两条记忆以"未完成"状态常驻在通知栏，大字显示码值。小林到驿站后，下拉通知栏，直接报出两个码，取完件后在通知上点"已取"，两条记忆自动标记为已完成，从通知栏消失。

全程没有打开任何 App，没有翻找任何短信。

---

**中午 12:00 — 点外卖**

小林在美团点了一份午餐。下单后他截了一张订单页面的图，从系统分享菜单直接分享到马良笔记。

App 在后台启动 AI 前台服务（`AiProcessingService`），通知栏显示"AI 识别中…"的进度条。2 秒后，通知变成：

> ✅ 已识别 2 条记忆：
> - 🍜 取餐码 **A032**（肯德基）
> - 💰 账单 **-¥32.50**（餐饮·美团外卖）

一张截图，AI 同时识别出了取餐码和消费金额，自动生成了两条不同类型的记录。取餐码以大字通知常驻，方便到店取餐；账单自动归类为"餐饮"，金额、商家、支付方式、时间全部自动填好。

小林到店后，下拉通知栏看到取餐码"A032"，报给店员，取完餐点"已取"。

---

**下午 2:00 — 逛商场买衣服**

小林在优衣库买了一件白色 T 恤，¥99。他拍了一张衣服的实物照片，分享到马良笔记。

AI 这次识别出这是一件服饰，自动提取了：
- 👕 服饰名称：白色圆领 T 恤
- 分类：T 恤
- 颜色：#FFFFFF（白色）
- 适用季节：春季、夏季
- 品牌：UNIQLO
- 价格：¥99
- 尺码：L

然后他又拍了一张小票，AI 同时生成了一条账单记录：
- 💰 账单 **-¥99.00**（服饰·UNIQLO）

小林的衣橱自动多了一件白色 T 恤，下次 AI 推荐穿搭时就会把这件新衣服纳入考虑。

---

**下午 4:00 — 咖啡店看到有用信息**

小林在咖啡店看到墙上贴着一张活动海报，上面有下周末市集的时间和地点。他随手拍了一张照片。

AI 识别后归类为"随手记"，自动提取了关键信息：
- 📝 标题：周末创意市集
- 摘要：3 月 22 日 10:00-18:00，静安嘉里中心广场
- 日程：自动识别出日期和时间，创建了日程提醒

小林不需要手动输入任何文字，AI 把海报上的信息结构化地整理好了。

---

**晚上 8:00 — 查看今日消费**

小林好奇今天花了多少钱，打开马良笔记首页，点击顶部的"总账单"合集卡片。

账单汇总页顶部，AI 财务助手以流式输出的方式，像朋友聊天一样说：

> "今天花了 ¥131.50，两笔——午饭 32.5，买衣服 99。这个月服饰支出已经到 ¥580 了，比上个月同期多了 30%，后面可以稍微控制一下 😊"

小林点了"问 AI"按钮，底部弹出一个带渐变光效的对话面板，自动进入语音监听模式。他问：

> "我这个月在吃饭上花了多少？"

AI 2 秒后回答：

> "这个月餐饮一共花了 ¥487，日均大概 ¥35。你最常点的是肯德基（6 次）和瑞幸（8 次），加起来占了餐饮支出的一半多。"

小林又打字问了一句："跟上个月比呢？"AI 基于上下文继续回答，不需要重复说明"餐饮"这个分类。

---

**晚上 9:30 — 整理记忆合集**

小林想把这周末的所有记忆整理到一起。他在首页的合集区域点击"+"，创建了一个新合集"周末日常"，把今天的取件码、外卖、买衣服、市集海报这几条记忆都加了进去。

以后回顾这个周末，打开合集就能看到完整的生活切片。

### 体验对比总结

| 维度 | 传统方式（多个 App 分散管理） | 马良笔记 AI 统一管理 |
|------|---------------------------|---------------------|
| 信息入口 | 分散在 5-10 个 App 中 | 统一入口：拍照/截图/分享 |
| 每条信息录入耗时 | 30-120 秒（手动输入） | 0-3 秒（AI 自动识别） |
| 取餐码/取件码查找 | 15-60 秒（翻找 App/短信） | 0 秒（通知栏常驻大字显示） |
| 一图多信息 | 不支持（一张截图只能存一个地方） | 支持（一张外卖截图同时生成取餐码+账单） |
| 记账分类 | 手动从 20+ 分类中选择 | AI 自动分类（22 种支出 + 8 种收入） |
| 衣橱管理 | 无（或需要专门的衣橱 App） | 拍照自动入库，AI 提取颜色/尺码/季节 |
| 穿搭建议 | 无 | AI 结合实时天气 + 衣橱数据推荐 |
| 消费洞察 | 月底手动看图表 | AI 实时分析 + 语音/文字对话查询 |
| 临时备忘 | 截图存相册，找不到 | AI 自动提取关键信息 + 日程识别 |
| 情感体验 | 分散、焦虑、操作负担重 | 统一、轻松、像有个贴心助手 |
